# 📚 Multimodal Study Assistant (Віртуальний асистент для навчання)

Цей проєкт розроблено в рамках виконання **КП 4 "Віртуальний асистент з використанням LLM"**. 
Обрана тема: **2. Multimodal Study Assistant**.

Це інтерактивний вебзастосунок на базі **Streamlit** та **Google Gemini API**, який допомагає студентам та учням вчитися ефективніше. Користувач може завантажити фото свого конспекту, уривок з підручника або скриншот екрана, а віртуальний асистент автоматично проаналізує його, виділить найголовніше, пояснить незрозумілі терміни та створить інтерактивний тест (квіз) для перевірки знань.

---

## ✨ Основні можливості (Features)

- 📸 **Мультимодальний аналіз**: Підтримка завантаження зображень (JPG, PNG). Асистент «читає» конспекти за допомогою Vision-можливостей моделі **Gemini 2.5 Flash**.
- 📝 **Виділення головного**: Автоматичне розпізнавання тексту та витяг ключових тез.
- 💡 **Словник термінів**: Пошук складних термінів у тексті та їх пояснення простою, зрозумілою мовою.
- 🎓 **Інтерактивний квіз**: Генерація мінімум 3-х тестових запитань з 4 варіантами відповідей на основі завантаженого матеріалу. Оцінка результатів та можливість генерувати нові запитання.
- 🇺🇦 **Локалізація**: Асистент спілкується та генерує відповіді виключно українською мовою.

---

## 🛠 Технологічний стек та архітектура

- **Мова програмування:** Python >= 3.13
- **Вебфреймворк:** [Streamlit](https://streamlit.io/) — для швидкої розробки зручного інтерфейсу користувача.
- **LLM інтеграція:** `google-genai` — офіційний SDK для взаємодії з Google Gemini API. Використовується швидка та потужна модель `gemini-2.5-flash`.
- **Структурований вивід (Structured Output):** [Pydantic](https://docs.pydantic.dev/) — використовується для опису схем даних (`StudyAnalysis`, `TermExplanation`, `QuizQuestion`), що гарантує отримання валідного JSON-об'єкта від LLM та його миттєву типізацію в Python-об'єкти.
- **Тестування:** `pytest` із застосуванням `unittest.mock.patch` для імітації (mock) відповідей від API та тестування обробки помилок.
- **Менеджер залежностей:** `uv` — сучасний, надшвидкий інструмент для керування віртуальним середовищем та залежностями.


## 🎯 Відповідність технічним вимогам КП 4

Проєкт повністю відповідає всім заявленим вимогам завдання:

1. **Обробка помилок (Exception Handling)**:
   - Використовуються блоки `try...except` для всіх викликів до API (див. `core.py`).
   - Коректно обробляються помилки перевищення лімітів `ResourceExhausted` (код 429) та невірних даних `InvalidArgument` (код 400), повертаючи зрозумілі повідомлення користувачеві.
2. **Типізація (Pydantic)**:
   - Використано функціонал `response_schema` при запиті до Gemini, що забезпечує стабільне повернення валідного JSON.
3. **Застосування `system_instruction`**:
   - Роль та правила поведінки моделі винесені в окремий параметр `SYSTEM_INSTRUCTION` (роль "Senior Developer" у контексті навчального помічника).
4. **Обґрунтовані параметри моделі**:
   - `temperature=0.7` — додає креативності при генерації цікавих та різноманітних тестових запитань.
   - `top_p=0.9` — утримує модель від занадто сильних відхилень від фактів, що важливо для навчального матеріалу.
5. **Тестування**: 
   - Написані модульні тести на `pytest`, що перевіряють логіку обробки успішних та помилкових запитів.

## 🚀 Встановлення та запуск

### 1. Попередня підготовка
Переконайтеся, що у вас встановлені:
- **Python** версії 3.13 або вище.
- **[uv](https://docs.astral.sh/uv/)** — швидкий менеджер пакетів та середовищ для Python.

### 2. Клонування репозиторію
```bash
git clone <url-вашого-репозиторію>
cd kp-4
```

### 3. Налаштування API Ключа
Для роботи програми потрібен безкоштовний ключ **Google Gemini API**. 
1. Отримайте ключ на [Google AI Studio](https://aistudio.google.com/).
2. Створіть файл `.env` у корені проєкту:
```bash
touch .env
```
3. Додайте у файл свій ключ:
```env
GEMINI_API_KEY="AIzaSyYourSecretKeyHere..."
```

### 4. Встановлення залежностей та запуск

Завдяки використанню `uv`, ви можете запустити програму без необхідності створювати віртуальне середовище вручну — `uv` зробить це автоматично!

Запустіть застосунок за допомогою команди:
```bash
uv run streamlit run src/assistant/main.py
```
*Після запуску у вашому браузері автоматично відкриється сторінка `http://localhost:8501` з інтерфейсом асистента.*

## 📘 Інструкція з використання (User Guide)

1. **Завантаження матеріалу:** Натисніть кнопку "Browse files" та оберіть зображення вашого конспекту (JPG/PNG).
2. **Аналіз:** Натисніть кнопку "Проаналізувати матеріал" і зачекайте кілька секунд.
3. **Вивчення результатів:** Ознайомтесь із розпізнаним текстом та поясненням складних термінів.
4. **Проходження квізу:** 
   - Відзначте варіанти відповідей у блоці "Перевірка знань".
   - Натисніть "Перевірити відповіді", щоб побачити свій результат.
5. **Подальші дії:** Після проходження ви можете повторити тест, згенерувати абсолютно нові питання для цього ж конспекту (кнопка "Нові питання") або завантажити інший файл.
